# セッション振り返り (Wrap-up)

会話を閉じる前に以下のチェックリストを順に実行し、結果をまとめて報告する。

## チェックリスト

```
Session Wrap-up:
- [ ] Step 1: 作業状態の確認
- [ ] Step 2: 残課題・TODOの洗い出し
- [ ] Step 3: メモリ化すべき知見の記録
- [ ] Step 4: サマリー出力
```

## Step 1: 作業状態の確認

```bash
git status -u
git log --oneline -5
git stash list
```

確認ポイント:
- 未コミットの変更がないか
- 未プッシュのコミットがないか
- stashに退避した作業が残っていないか

問題があればユーザーに報告し、対応を確認する。

## Step 2: 残課題・TODOの洗い出し

会話中に以下に該当するものがなかったか振り返る:
- 着手できなかったタスク
- 一時的にスキップした対応（「後で」「次回」など）
- 発見したが修正しなかったバグや改善点

該当があれば、GitHub Issue化するかメモに残すかユーザーに確認する。

## Step 3: メモリ化すべき知見の記録

以下に該当する知見がセッション中にあったか振り返る:
- ハマったポイントとその解決策
- 発見したプロジェクト固有のパターンや制約
- ツールやAPIの使い方で学んだこと
- ユーザーのワークフローに関する新しい好み

該当があれば `memory/` 配下の適切なファイルに記録する。
- 既存ファイルに追記できる場合は追記
- 新規トピックの場合はファイルを作成し、MEMORY.md からリンク

## Step 4: サマリー出力

以下のフォーマットで報告する:

```
## Session Wrap-up

### 作業状態
- (クリーン or 未コミット変更あり / 未プッシュあり / stashあり)

### 今回やったこと
- 実施内容を箇条書き

### 残課題
- (なし or 課題リスト + 対応方針)

### メモリ更新
- (なし or 更新したファイルと内容の要約)
```
